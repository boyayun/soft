Introduction

This document describes the Freescale® peripheral bare-metal drivers for Kinetis® M series of microcontrollers. The drivers are provided in source code. The source code files are intended to be included directly into the application project or built separately into a statically linked library. This document gives you all information about bare-metal drivers' configuration and their use in an application. The Directory Structure explains the source files and header files important for your application. The Drivers FAQ is available to answer the most common questions. Please read the License document for terms and conditions under which you can use the software.

CREATING NEW PROJECT
Execute make_project.exe script from the "template" subfolder to create a new project. The script creates a new project based on selected toolchain, debugger probe, device derivative and project name. The new project can be either standalone or part of drivers' installation. The standalone project will be created in a specific subfolder and it will contain snapshoot of all source files necessary to build application.

BASIC DATA TYPES
The drivers define some basic data types to support code portability between different hardware architectures and tools. These Basic Data Types definitions can be found in the common.h header file.

CORE'S PLATFORM INTERFACE
The arm_cm0.h header file contains constants, macros and inline functions which interface to the Kinetis-M core's platform architecture based on the ARM® Cortex™-M0+ core. Refer to Core's Platform Interface for more details.

BASIC I/O OPERATIONS
The header iohw.h declares several Basic Operations for making direct access to I/O registers supported by core platform's Bit Manipulation Engine (BME). The BME provides robust read-modify-write capability to low-end Kinetis-M microcontrollers and improves software drivers' Code Density.

PERIPHERAL DRIVERS
Freescale's Kinetis-M devices are integrated with powerful 24-bit Sigma Delta analogue to digital converter, low-noise Programmable Gain Amplifier, timers, communication interfaces and additional on-chip peripherals. EXAMPLES
Software Examples demonstrate correct use of the Kinetis-M peripheral bare-metal drivers in an application. Each peripheral driver is complemented by one or more software examples depending on the driver complexity and number of features to be demonstrated.

BOOT MODES
Freescale's Kinetis-M microcontroller series supports several boot modes. The preferred boot mode can be activated by constants in the appconfig.h file. For more information about boot time and current consumption profile for each supported boot mode refer to Boot Modes.
